## How To Play (MVP)

1. Create an adventurer.
2. Move to an adjacent hex and discover that hex.
3. Discover an area in that hex.
4. Initialize harvesting, then start and complete harvesting.
5. Convert harvested items into energy.
6. Pay hex maintenance to prevent decay.
7. Defend your territory, or claim a decayed hex from another player.

## Gameplay Loop Summary

`explore -> discover -> harvest -> convert -> maintain -> defend/claim -> repeat`

## Known Limitations

- This release targets the MVP loop only; deeper systems (for example advanced mining/crafting/buildings UX) are outside this player runbook.
- Actions are adjacency-gated and energy-gated.
- Permadeath is active for adventurers.
- Slot Katana currently requires explicit high `L2Gas` bounds for `sozo migrate/execute` on this world size.

## Troubleshooting

- If transaction submission fails with `InsufficientResourcesForValidate` or `Insufficient max L2Gas`, raise `--l2-gas` and retry.
- If you see `Missing latest block number` on optimistic setups, create a fresh Katana with `--config` (non-optimistic) or switch fork provider.
- If reads fail, confirm `world_address` matches `game/manifest_dev.json` from the latest migration.
- If Slot API calls fail, verify `slot auth info` and the target team/project access.
